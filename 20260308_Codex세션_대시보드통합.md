## 보충: 대화 중 검토 내용

### Codex 세션 포맷 분석

Codex 세션은 `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl` 구조로 저장. JSONL 각 줄은 `{timestamp, type, payload}` 3-레벨 구조 (Claude의 플랫 구조와 다름).

주요 이벤트 타입:
- `session_meta`: 세션 메타 (id, cwd, originator, git, cli_version)
- `turn_context`: 턴별 컨텍스트 (model, sandbox_policy, approval_policy)
- `response_item`: 메시지/추론/도구호출 (`payload.type`으로 세분화)
- `event_msg`: 이벤트 (task_started, task_complete, token_count 등)

### 의사결정 근거

**데이터 정규화 전략 채택**: build.js에서 Codex JSONL을 Claude와 동일한 `{role, text, timestamp, tools}` 포맷으로 변환. index.html 메시지 렌더링 코드 변경을 최소화하기 위함.

**codex_exec 포함 방식**: 사용자가 "핵심 대화를 기본으로, full 버전은 선택 가능하게"를 요청. → Codex 탭에 "서브태스크 포함" 체크박스로 구현. 기본 OFF(CLI만), ON 시 Exec도 포함.

### 검토하고 제외한 대안

| 대안 | 제외 이유 |
|------|-----------|
| codex_exec 완전 제외 | 사용자가 plan 결과물(exec)도 보고 싶다고 명시 |
| 별도 Exec 탭 추가 | 탭이 5개로 너무 많아짐. 체크박스 토글이 더 깔끔 |
| CLI/Exec 배지로만 구분 | 기본 상태에서 exec가 대량 노출되어 핵심 대화 묻힘 |
