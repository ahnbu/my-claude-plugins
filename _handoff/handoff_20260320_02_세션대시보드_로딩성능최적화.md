# 세션 대시보드 로딩 성능 최적화

## 완료된 작업

### Phase 1: Gemini 캐시 버그 수정 (`shared/session-db.js`)
- 필터 세션(auto-call, <60초)을 `type='gemini_excluded'` sentinel 레코드로 DB에 저장 → 재파싱 방지
- 디버그 로그 추가 (각 파일별 캐시/미스 이유 출력)
- `getAllMeta()`에서 `gemini_excluded` 제외 필터 추가
- 실측: 캐시 40→82 (sentinel 42개 동작)

### Phase 2: 데이터 분리 + Lazy Load
- `build.js`: 메시지를 세션별 `data/*.json` 파일로 분리 출력 (2,096개)
- `index.html`: `loadConversation()` → async fetch 기반, `messageCache` 도입
- `serve.js`: `data/*.json` 정적 서빙 (경로 순회 방지)
- `build.js`: 증분 빌드 — DB mtime vs 파일 mtime 비교 + `utimesSync` + 고아 파일 삭제
- safeId에서 `:` 제거 (Windows 파일명 호환)
- 실측: HTML 99MB→33MB, 증분 2회차 스킵 2090/갱신 2

### Phase 3: Progressive Rendering (`index.html`)
- 가상 스크롤 (ITEM_HEIGHT=82, BUFFER=5, requestAnimationFrame)
- 이벤트 위임 (인라인 onclick → 컨테이너 리스너)
- `selectSession()` / `showPlanDetail()` / `copyDoc()` → async 변환
- "더보기" 50개 제한 제거 (전체 메시지 렌더링 복원)

### 전문 검색 인덱스 확장
- 검색 인덱스: `substring(0, 2000)` 제한 제거
- 도구 JSON (46MB/69%) 제외, `m.text`만 전체 포함
- HTML 33MB (99MB 대비 67% 감소)

## 미해결 이슈

- **Gemini 동일 UUID 복수 파일**: 매번 "신규 17" 잔존 → `totalNew > 0` → 빌드 스킵 불가. 증분 빌드로 I/O는 최소화됨. 근본 해결에는 Gemini 파싱 로직 재설계 필요 (범위 밖).
- **브라우저 실측 미완**: `/ss` 실행 → 대시보드 표시까지 3초 이내 확인 필요
