| 단계 | 상태 | 산출물 |

| Phase 3: 스킬 디렉토리 정리 | ✅ 완료 | timeline/transcript 삭제, analyzer 보류, 테스트 19/19 통과 |
| 온디맨드 타임라인 버튼 | ⬜ 미착수 | 후속 작업으로 분리 |

### 핵심 의사결정 로그
- [결정] session-analyzer → 보류 (코드 유지, 배포 안 함). 이유: 디버그 로그 기반 bash 스크립트로 Windows 불안정 + 사용 시나리오 부재
- [결정] session-transcript → 기법만 대시보드에 흡수 후 폐기. 이유: shortenToolName, toolSummary 폴백, 도구 응답 필터 등 유용 함수만 이식
- [결정] session-timeline → 하이브리드 통합 (요약 메트릭은 DB, 상세는 향후 온디맨드). 이유: tool_count/error_count는 DB에 이미 존재 또는 추가, 상세 타임라인은 구현 복잡도 고려 후속으로 분리
- [결정] skill-doctor/skill-creator-my 통합 불가. 이유: doctor는 통계(운영), analyzer는 검증(QA), creator는 생성(개발) — 관심사가 다름

### 다음 세션 시작점
- 온디맨드 타임라인 버튼 구현: 대시보드 세션 상세 패널에 "타임라인 보기" 버튼 추가 → 클릭 시 lib/session/timeline-builder.js로 계산 → 모달 표시
