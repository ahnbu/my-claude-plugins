## 1. 현재 상태

### 작업 목표
`my-claude-plugins` 레포를 `~/.claude/`로 완전 통합하여 스킬·훅·DB를 1곳에서 관리. 레포는 `my-claude-plugins-archive`로 이름 변경하여 보존.

### 진행 현황
| 단계 | 상태 | 산출물 |
|------|------|--------|
| 현황 분석 + 관련 문서 탐색 | ✅ 완료 | `_docs/공통/20260310_*` 문서 확인 |
| 옵션 비교 분석 (A/B/C) | ✅ 완료 | 대화 중 비교표 작성 |
| 결정사항 확정 (사용자 확인) | ✅ 완료 | Option B (완전 통합) 채택 |
| 전수 참조 조사 (explore agents) | ✅ 완료 | 코드 파일 참조 30+곳 파악 |
| 마이그레이션 실행 계획 작성 | ✅ 완료 | `plans/warm-bouncing-puzzle.md` |
| **실행 (Wave 1~9)** | ⬜ 미착수 | - |

### 핵심 의사결정 로그
- [결정 1] **Option B (완전 통합)** 채택. 인지 부하가 코드 변경 빈도와 무관하게 매 세션 누적되는 탐색 비용이라는 점에서 1회 마이그레이션이 정당화됨
- [결정 2] **세션DB → `~/.claude/skills/shared/sessions.db`**. D드라이브 크로스 의존 회피, `__dirname` 상대경로 활용 가능 (`session-db.js`와 같은 폴더)
- [결정 3] **대시보드 웹앱 → `D:\vibe-coding\session-dashboard\`**. 스킬은 `~/.claude/skills/session-dashboard/`에 그대로. 웹앱만 이동
- [결정 4] **아카이브 = 폴더명 변경만** (`my-claude-plugins-archive`). 내용물 전체 보존, 연결지점 끊음
- [결정 5] **보일러플레이트 별도 제거 불필요** — 아카이브 통째로 보존하면 해결
