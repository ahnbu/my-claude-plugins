# cp·wrap 스킬 커밋 로직 스크립트화 개선 2차

## 발단: 사용자 요청

이전 세션(doc: `20260327_02`)에서 cp·wrap 스킬 개편이 완료됐으나, feed-my 검토 결과 미완 사항이 확인됨. 이번 세션에서 추가 개편 및 pre-commit hook 통합 작업 진행.

## 작업 상세내역

### 1. wrap 스킬 현황 재검토 (feed-my)

구 wrap(`SKILL_wrap_20260327.md`) vs 신 wrap 비교:

- cp 위임 방향 ✅
- 소실된 기능 3개 확인:
  - 관심사별 멀티커밋 (구 Step 3-2)
  - `_handoff/` 별도 커밋 (구 Step 3-3)
  - pre-commit hook 경고 (구 Step 3-2 ⚠️)

### 2. 관심사별 멀티커밋 분석

| 항목 | 현재: 레포당 1커밋 | 변경: 관심사별 멀티커밋 |
|---|---|---|
| pre-commit hook 안전성 | ✅ 항상 통과 | ⚠️ 순서 틀리면 차단 |
| git log 가독성 | ████░░ | ██████ ✅ |
| 분류 일관성 | ✅ 판단 없음 | ❌ Claude 매번 판단 |
| 단독 /cp 적합성 | ✅ 경량 취지 맞음 | ⚠️ 오버킬 가능 |

**결론**: 글로벌 CLAUDE.md "한 커밋 = 한 관심사" 규칙이 이미 커버 → cp 스킬에 명시 불필요. 실제 실행 결과(이전 세션 doc)에서도 관심사별 분리가 자연스럽게 동작 확인.

### 3. pre-commit hook 통합

**문제**: my-claude-plugins 레포에 `core.hooksPath = git-hooks/` 설정 → 글로벌 hook 무시. 두 파일 중복 유지보수.

**결정**: 옵션 A 선택 — 글로벌 hook이 공통 CHANGELOG 규칙을 담당하고, repo 전용 스크립트(`git-hooks/check-session-db-doc.js`)는 파일 존재 시 자동 호출.

**변경**:
- 글로벌 hook(`~/.config/git/hooks/pre-commit`) 재구조화: if/elif/else로 통합, 레포 전용 스크립트 자동 감지 블록 추가
- `core.hooksPath` 제거 → 글로벌 hook이 my-claude-plugins에도 적용
- `git-hooks/pre-commit` trash
- `check-version-sync.js.bak` 호출 제거 (플러그인 dead 상태 — 버전 동기화 검증 불필요)

**글로벌 hook 구조**:
```
1. CHANGELOG 검증 (공통)
   - _handoff/ 전용 커밋은 스킵
   - 서브폴더 CHANGELOG 차단
   - staged / 없으면 auto-create / 없고 staged 안 됨 → 차단
   - [여러 커밋으로 분리할 때 필수 순서] 안내 포함
2. 레포 전용 검증 (git-hooks/ 스크립트 자동 감지)
   - check-session-db-doc.js 있으면 실행
```

### 4. cp 스킬 Step 2-4 멀티커밋 경고 추가

글로벌 hook 문구를 그대로 인용:

```
> ⚠️ 여러 커밋으로 분리할 때 필수 순서
> 커밋1: CHANGELOG에 커밋1 내용만 기록 → git add CHANGELOG.md + 소스 → git commit
> 커밋2: CHANGELOG에 커밋2 내용만 추가 기록 → git add CHANGELOG.md + 소스 → git commit
> ※ CHANGELOG를 여러 커밋분 한꺼번에 수정하면 이 hook에서 차단된다.
```

### 5. changelog-add-row.mjs CHANGELOG auto-create

**문제**: CHANGELOG.md 없으면 exit 1. cp 스킬에 존재 확인 + 조건부 실행 2단계 필요.

**변경**: 스크립트 자체에서 CHANGELOG 없으면 템플릿 auto-create 후 행 삽입.

```diff
- process.stderr.write(`CHANGELOG.md를 찾을 수 없습니다: ${changelogPath}\n`);
- process.exit(1);
+ // 템플릿에서 자동 생성 후 계속
```

cp 스킬 Step 2-4도 `ls ... && echo EXISTS` 체크 제거 → 스크립트 1회 호출로 단순화.

### 6. cp 스킬 구조 개선

**Step 2-1 → Step 1 통합**: `status --short`, `diff --stat`, `diff --cached --stat`를 Step 1 레포 감지 단계로 이동 (status --porcelain 중복 제거).

**Step 3+4 통합**: 에러 처리 + 결과 출력을 Step 3으로 합침.

**최종 구조**:
```
Step 1: 레포 감지 + 상태 파악
Step 2: 레포별 처리
  2-1. Staging
  2-2. 커밋 메시지 생성
  2-3. CHANGELOG 업데이트
  2-4. Commit + Push
Step 3: 결과 출력
```

## 의사결정 기록

### pre-commit hook 통합 방식

- 결정: **옵션 A** (글로벌 + repo 스크립트 자동 감지)
- 근거: 글로벌 hook이 my-claude-plugins 경로를 하드코딩하지 않아도 됨. 다른 레포에도 재사용 가능.
- 트레이드오프: 파일 수 약간 증가 vs 글로벌 hook의 순수성 유지

### check-version-sync.js.bak 제거

- 결정: 호출 제거 (파일 보관)
- 근거: 플러그인이 dead 상태(로컬 전용). 배포 목적 없으면 버전 동기화 강제 불필요.

### Step 1.5 (Wrap 체크리스트) 이미 제거됨

- 이전 세션에서 의도적 삭제. 95% 사용 안 함.
