### 신규 스크립트 생성 vs next-handoff.mjs 확장

| | 신규 `wrap-context.mjs` 생성 | `next-handoff.mjs` 확장 |
|---|---|---|
| 파일 수 | +1개 (관리 대상 증가) | 기존 파일 유지 |

| 호환성 | 독립적, 깔끔한 인터페이스 | 기존 호출부 영향 가능 |
| **채택** | | ✅ **확장 방향 채택** |

- **결정**: `next-handoff.mjs` 확장 (신규 스크립트 생성 취소)
- **근거**: 중복 구현 방지, 이미 안정화된 로직 재사용
- **트레이드오프**: next-handoff.mjs가 비대해질 수 있으나, 동일 책임 범위 내 확장이므로 허용

### doc-save-cp Common Mistakes 처리

- **결정**: Common Mistakes 섹션 전체 삭제 + 핵심 경고 1개만 When to Use에 흡수
- **근거**: 중복 제거로 스킬 간결화, 핵심 경고(순서 역전)만 보존
- **트레이드오프**: 명시적 경고 목록 소실 → When to Use가 이를 보완

### Phase 2: 신규 스크립트 생성

#### 2-1. `scripts/wrap-context.mjs` (Step 0+1+2-1 통합)

wrap 실행 시 첫 호출. 기계적 컨텍스트를 한 번에 수집하여 JSON 출력.

```
입력: [--session-id <id>]
출력 (JSON):
{
  runtime: "claude"|"codex"|"gemini"|"unknown",
  session_id: "<id>",
  session_path: "<path>",
  git_available: true|false,
  repos: [{ root, files: [...], file_count }],
  non_repo_files: [...]
}
```

구현 내용:
- 런타임 감지: 환경변수 4개 체크 (`CLAUDECODE`, `CODEX_THREAD_ID`, `GEMINI_CLI`, `CLAUDE_CODE`)
- 세션 ID: 인자 > 환경변수 > `.claude/.current-session-id` fallback
- 세션 경로: 런타임별 경로 패턴 조합
- Git 감지: `git status --short` → `git diff --name-only HEAD` + `git diff --name-only --cached` + `git ls-files --others --exclude-standard` → 레포 루트 그룹핑

이렇게 하면 SKILL.md에서 Step 0(13줄) + Step 1(29줄) + Step 2-1(35줄) = **77줄을 스크립트 호출 ~8줄로 대체**.

#### 2-2. `scripts/wrap-verify-gaps.mjs` (Step 2-4)

```
입력: --session-id <id> --handoff <path>
출력 (JSON):
{
  missing_decisions: [...],
  missing_unresolved: [...],
  missing_lessons: [...],
  all_covered: true|false
}
```

`extract-session-summary.js`의 gaps 배열을 handoff 텍스트와 keyword 매칭하여 누락 항목 식별.
