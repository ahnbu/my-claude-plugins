실제 데이터 확인 결과: 73개 파일 중 7개 sessionId가 2~3개 파일에 걸쳐 중복 저장됨(자동 저장 체크포인트).

| 옵션 | 방식 | 결과 |
|------|------|------|
| 중복 파싱 방지 | 사전에 UUID → 파일 목록 맵 빌드 후 winner만 처리 | ⚠️ 파일을 먼저 모두 파싱해야 UUID를 알 수 있어 복잡 |
| ✅ INSERT OR REPLACE 수용 | 사전순 정렬 → 마지막(최신) 파일이 자동 승리. 이전 파일은 매 sync마다 재파싱(harmless) | 단순하고 결과 동일. 재파싱 비용 미미 |

- 결정: INSERT OR REPLACE 수용 방식
- 근거: 동일 UUID 파일 수 = 8개(9개 캐시 미스 중 8개). 파싱 비용 < 구현 복잡도
- 트레이드오프: 매 build.js 실행 시 8개 파일이 "신규"로 표시됨(실제 변경 없음). 기능 정확성에는 영향 없음

## 단계 1: session-parser.js — Gemini 파서 추가

### 1.1 processGeminiSession(filePath, projectRoot)

JSON.parse로 파일 읽기 (JSONL이 아닌 단일 JSON).

**필드 매핑:**

| DB 컬럼 (metadata 키) | Gemini 소스 |
|----------------------|-------------|
| sessionId | `"gemini:" + data.sessionId` |
| type | `"gemini"` |
| timestamp | `data.startTime` |
| lastTimestamp | `data.lastUpdated` |
| models | gemini 메시지의 `msg.model` 수집 (Set → 배열) |
| userEntryCount | `type === "user"` 메시지 개수 |
| userTextMessageCount | 위와 동일 (Gemini에 isMeta 개념 없음) |
| toolUseCount | 모든 gemini 메시지의 `toolCalls.length` 합산 |
| toolResultCount | toolUseCount와 동일 (Gemini는 호출+결과 통합) |
| totalInputTokens | Σ `(tokens.input + tokens.cached + tokens.tool)` |
| totalOutputTokens | Σ `(tokens.output + tokens.thoughts)` |
| toolNames | `{ [tc.name]: count }` 맵 |
| firstMessage | 첫 user 메시지 content[0].text (200자) |
| project | 인자로 받은 projectRoot |
| gitBranch | `""` |
| keywords | `extractKeywords(firstMessage)` |
| title | `formatTimestamp(timestamp) + "_" + keywords.join("_")` |
| filePath | 절대 경로 |

**messages 배열 구축:**
- `msg.type === "user"` → `{ role: "user", subtype: "user_input", text: content[0].text, timestamp }`
- `msg.type === "gemini"` → `{ role: "assistant", text: msg.content, timestamp, tools: toolCalls 매핑 }`
- `msg.type === "error"` → 건너뜀

### 1.2 normalizeGeminiEntries(rawMessages)

session-*.json의 원본 messages 배열 → events 테이블용 정규화.

| Gemini 소스 | events.kind |
|-------------|-------------|
| `type: "user"` + content | `user_text` |
| `type: "gemini"` + content | `assistant_text` |
| `type: "gemini"` + thoughts[] | `assistant_thinking` |
| toolCalls[] (각 호출) | `tool_use` |
| toolCalls[].result | `tool_result` |

### 1.3 exports에 추가

```js

```

## 엣지 케이스

1. **동일 sessionId 다중 파일**: 같은 세션이 여러 session-*.json에 존재 가능 (중간 저장). `fs.readdirSync`는 정렬 미보장이므로, chats/ 내 파일 목록을 **파일명 사전순 정렬** 후 처리 (파일명에 ISO 타임스탬프 포함 → 사전순=시간순). `INSERT OR REPLACE`로 마지막(=최신) 파싱 결과가 자동 승리.
2. **빈 세션**: `userEntryCount === 0` (error만 있는 세션) 건너뜀.
3. **user content 배열**: `msg.content`가 `[{text: "..."}]` — `.map(c => c.text).join("\n")`.
4. **gemini content 문자열**: `msg.content`가 plain string.
5. **toolCalls 미존재**: optional chaining (`msg.toolCalls?.length`).
6. **thoughts 빈 배열**: `[]`이면 무시.
7. **JSON 파싱 오류**: try-catch + warn.
