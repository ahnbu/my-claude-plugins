### 분석: Q1 — `userEntryCount` 이름 적절성

**결론: 이름 유지, 코드 주석 보완**

- JSONL `entry.type === "user"` = Claude API상 user role 전체 → 프로토콜 기준 정확
- `toolResultCount`는 user entry 내 `content[]`의 `tool_result` 블록 수 → 부분집합 관계

```
userEntryCount = userTextMessageCount + tool_result 전용 entry + isMeta entry + 기타
```

### 구현 결과

#### Step 1: `shared/text-utils.js` — `task-notification` 등록

두 regex 패턴에 추가:
```
command-message|...|task-notification|antml:\w+
```

#### Step 2: `shared/session-parser.js` — isMeta 제외 + subtype 분류

```javascript
// 변경 전: cleanText.trim()이면 무조건 카운트
if (cleanText.trim()) { userTextMessageCount++; }

// 변경 후: isMeta 제외
const isUserText = !entry.isMeta && cleanText.trim();
if (isUserText) { userTextMessageCount++; }

// subtype 분류
const subtype = entry.isMeta ? "meta"
  : (Array.isArray(content) && content.some(b => b.type === "tool_result")) ? "tool_result"
  : "user_input";

messages.push({ role: "user", subtype, text: cleanText, ... });
```

`processCodexSession`에도 `subtype: "user_input"` 추가.

#### Step 3: `my-session-dashboard/index.html` — 사이드바 도구 수 제거

```javascript
// Before
: `<span>${s.userTextMessageCount || 0} 메시지</span><span>${s.toolUseCount || 0} 도구</span>`;
// After
: `<span>${s.userTextMessageCount || 0} 메시지</span>`;
```

#### Step 4: `my-session-dashboard/index.html` — 대화 뷰어 라벨 분화

```javascript
const roleLabel = msg.role === "user"
  ? (msg.subtype === "meta" ? "스킬"
    : msg.subtype === "tool_result" ? "도구 응답"
    : "사용자")
  : ROLE_LABELS[msg.role] || msg.role;

const roleBadgeClass = msg.role === "user"
  ? (msg.subtype === "meta" ? "meta"
    : msg.subtype === "tool_result" ? "tool-result"
    : "user")
  : msg.role;
```

CSS 추가:
```css
.role-badge.meta { background: #8b5cf644; color: #a78bfa; }
.role-badge.tool-result { background: #f59e0b33; color: #fbbf24; }
```

## 의사결정 기록

| 항목 | 선택지 | 결정 | 근거 |
|------|--------|------|------|
| `userEntryCount` 이름 | 변경 vs 유지 | **유지** | API 프로토콜 기준 정확, UI 미노출 내부 메트릭 |
| isMeta 제외 방식 | tag 등록 vs 플래그 검사 | **플래그 검사** (`!entry.isMeta`) | 직접적이고 명확, 누락 위험 없음 |

| 사이드바 도구 수 | 유지 vs 제거 | **제거** | 실질적 활용 없음, 메시지 수로 충분 |
| subtype 필드 | role 세분화 vs 별도 필드 | **별도 `subtype` 필드** | API role 의미 보존 + UI 관심사 분리 |
