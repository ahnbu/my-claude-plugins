# 대시보드 온디맨드 타임라인 구현 + UI 정리

## 발단: 사용자 요청

이전 세션(handoff `20260323_01_세션분석스킬3종_대시보드통합`)에서 미착수로 남긴 "온디맨드 타임라인 버튼" 구현을 이어서 진행. 대시보드 세션 상세 패널에서 타임라인을 온디맨드로 조회할 수 있는 기능 요청.

## 작업 상세내역

### Phase 1: 아키텍처 조사 및 접근법 결정

기존 대시보드 구조를 Explore 에이전트로 조사:
- 대시보드는 **완전 정적 HTML** — 빌드 타임에 `__SESSIONS_META__` + `__SESSIONS_DATA__`를 인라인 삽입
- 런타임 API 없음, `serve.js`는 SSE(빌드 진행도) + 정적 HTML 서빙만 담당
- 타임라인 계산은 JSONL 파싱이 필요하여 모든 세션 빌드 타임 처리는 비현실적

**결정**: `serve.js`에 API 엔드포인트 추가 → 프론트엔드 온디맨드 fetch → 모달 표시

### Phase 2: Timeline API 구현 (`serve.js`)

- `/api/timeline/:sessionId` 엔드포인트 추가
- 파이프라인: `loadSessionBundle` → `normalizeSessionBundle` → `buildTimeline`
- 의존성은 lazy load (`getTimelineDeps()`) — 첫 요청 시에만 로드
- 빌드 후 5초 자동 종료 → **5분 idle timeout**으로 변경 (API 요청 시 리셋)

### Phase 3: 타임라인 모달 UI (`index.html`)

- 세션 상세 헤더에 "타임라인" 버튼 추가 (plan 타입 제외)
- 모달 구성: 요약 통계 카드 5개 + 도구 호출 테이블 + 서브에이전트 테이블
- ESC 키 모달 닫기, 오버레이 클릭 닫기 지원

### Phase 4: 사용자 피드백 반영

1. **"소요" 열 추가** — 도구 호출 테이블에 "대기" 왼쪽에 "소요" 열 추가. 현재 도구 시작 → 다음 도구 시작까지의 간격 (AI 사고 + 도구 실행 포함)
2. **도구 수/에러 수 배지 제거** — 왼쪽 패널에서 `🔧 60 ❌ 2` 표시 제거 (상세 목록에서 확인 가능하므로 불필요)
3. **키워드 태그 제거** — 왼쪽 패널 세션 항목에서 키워드 태그(`read`, `follow`, `skill` 등) 제거

## 의사결정 기록

### 타임라인 데이터 제공 방식

| 방식 | 장점 | 단점 |
|------|------|------|
| 빌드 타임 전체 계산 | 서버 불필요 | 모든 세션 처리 → 빌드 시간 증가 |
| **serve.js API (채택)** | 온디맨드, 빌드 무영향 | 서버 실행 필요 |
| 브라우저 JS 포팅 | 서버 불필요 | 코드 중복, JSONL 접근 불가 |

- 결정: serve.js API 방식
- 근거: 대시보드는 이미 serve.js를 통해 열리므로 서버 의존성이 새로 생기는 게 아님. 빌드 성능 무영향.

### 서버 종료 전략

- 기존: 빌드 후 5초 자동 종료
- 변경: 5분 idle timeout (API 요청 시 리셋)
- 근거: API 엔드포인트 제공을 위해 서버 유지 필요. 무한 유지 대신 idle timeout으로 리소스 관리.
