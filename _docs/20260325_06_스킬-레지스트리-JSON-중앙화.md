# 스킬 레지스트리 JSON 중앙화

## 작업 상세내역

### 문제 상황

스킬 목록 데이터를 소비하는 2곳이 독립적으로 파일시스템을 스캔:

| 소비자 | 언어 | 스캔 대상 | 스캔 시점 |
|--------|------|----------|----------|
| `skill-doctor.py` | Python | 6개 경로 | 매 실행 |
| `session-parser.js` `buildCodexSkillWhitelist()` | JS | `~/.codex/skills/` 1개 경로 | 매 Codex 세션 파싱 |

결과: 동일 데이터 중복 수집, 새 스킬 추가 시 whitelist 갱신 누락 리스크, 소비자 추가 시 스캔 로직 재구현 필요.

### 구현 내역

**1. `scripts/build-skills-registry.mjs` 신규 생성**

6개 경로를 스캔하여 `~/.claude/skills/skills-registry.json`을 생성하는 스크립트:
- 스캔 순서: claude → codex → gemini (native, selfMade=true) → commands (selfMade=true) → plugin cache (selfMade=false) → my-plugins (selfMade=false)
- skill-doctor.py의 ALIASES 맵 동일하게 적용 (canonical name 해석)
- 각 스킬마다 SKILL.md frontmatter에서 description/context/model 파싱
- `upsertSkill()` 패턴: 첫 번째 등록이 selfMade 결정 (native > plugin 우선순위)
- 결과: 150개 스킬 등록

JSON 스키마:
```json
{
  "generated": "ISO timestamp",
  "skills": { "canonical-name": { name, description, sources, location, selfMade, context, model } },
  "aliases": { "old-name": "new-name" },
  "names": ["sorted canonical names array"]
}
```


효과: Codex 세션에서 전체 150개 스킬 화이트리스트 적용 (기존 codex 전용 스킬만 → 전체 통합)

**4. `hooks.json` — SessionStart hook 추가**

```json
"SessionStart": [{
  "hooks": [{
    "type": "command",
    "command": "node \"C:/Users/ahnbu/.claude/skills/scripts/build-skills-registry.mjs\"",

```

SessionStart 선택 이유: Stop hook은 context limit 종료 시 실행 미보장. SessionStart는 어떤 종료 방식이든 다음 세션에서 확실히 실행됨. 성능 영향 없음 (~100ms 이내).

## 의사결정 기록

### 레지스트리 형식 비교

| 대안 | 장점 | 단점 | 판단 |
|------|------|------|------|
| **A. JSON 레지스트리 (채택)** | 단일 소스, 언어 무관, 빠른 로드 | 갱신 시점 관리 필요 | 갱신 빈도 낮아 관리 부담 미미 |
| B. JS 모듈 | session-parser 직접 import 가능 | Python(skill-doctor)에서 사용 불편 | 크로스 언어 지원 어려움 |
| C. DB 테이블 | 세션DB 쿼리와 통합 | 과잉 설계, 스킬 목록은 정적 데이터 | 불필요한 복잡성 |

### 자동 갱신 방식 비교

| 방안 | 갱신 보장 | 커버리지 | 판단 |
|------|---------|---------|------|
| **SessionStart hook (채택)** | context limit 종료에도 안전 | 생성/삭제/리네임/통합 모두 | 다음 세션 시작 시 확실히 반영 |
| Stop hook | context limit 시 누락 가능 | 동일 | 보장 불확실 |
| PostToolUse (Write/Edit) | 즉시 | SKILL.md 생성/수정만 | 삭제/리네임 누락 |

- **결정**: JSON 레지스트리 + SessionStart hook 자동 갱신
- **근거**: 크로스 언어 소비자(Python + JS) 모두 지원. 파일 읽기 1회로 스캔 대체. 갱신은 세션 경계에서만 필요하므로 SessionStart가 최적.
- **트레이드오프**: 같은 세션 내 스킬 변경은 다음 세션에서 반영. 실제 사용에서 문제 없음 (스킬 변경 후 재사용은 대부분 다른 세션).

**목표**: 단일 JSON 레지스트리를 생성하는 스크립트를 만들어, 모든 소비자가 이 파일을 참조하도록 한다.

## 설계

### 레지스트리 파일

- **경로**: `~/.claude/skills/skills-registry.json`
  - 이유: 스킬 레포 루트에 위치하여 직관적. git 추적 가능. Python/JS 모두 절대경로로 접근.
- **생성 스크립트**: `~/.claude/skills/scripts/build-skills-registry.mjs`

### JSON 스키마

```json
{
  "generated": "2026-03-25T12:00:00Z",
  "skills": {
    "capture": {
      "name": "capture",
      "description": "제작자 중심 자료수집 오케스트레이터...",
      "sources": ["claude", "codex", "gemini"],
      "location": "claude",
      "selfMade": true,
      "context": null,
      "model": null
    },
    "superpowers:brainstorming": {
      "name": "superpowers:brainstorming",
      "description": "...",
      "sources": ["claude"],
      "location": "plugin:superpowers",
      "selfMade": false,
      "context": null,
      "model": null
    }
  },
  "aliases": {
    "my-session-wrap": "wrap",
    "plan-save": "doc-save"
  },
  "names": ["capture", "codex-my", "doc-save", "..."]
}
```

- `skills`: canonical name → 메타데이터 맵
- `aliases`: 구 이름 → canonical name (skill-doctor의 ALIASES 맵 통합)
- `names`: canonical name 배열 (화이트리스트 용도, Set 변환 간편)

### 소비자별 전환 계획

#### 3. 레지스트리 자동 갱신 — SessionStart hook

**변경 파일**: `~/.claude/my-claude-plugins/my-session-wrap/hooks/hooks.json`

- `SessionStart` 배열에 `build-skills-registry.mjs` 실행 hook 추가
- **SessionStart를 선택한 이유**: Stop hook은 context limit 초과로 세션이 종료될 때 실행이 보장되지 않음. SessionStart는 어떤 종료 방식이든 다음 세션에서 확실히 실행됨.
- 성능: 6개 경로의 디렉토리 읽기 + SKILL.md 존재 확인 = 동기 I/O ~200회, **100ms 이내** 완료. 세션 시작 체감 영향 없음.
- 트레이드오프: 스킬 변경 후 같은 세션 내에서는 미반영 → 다음 세션 시작 시 반영. (Stop hook도 같은 세션 내 미반영이므로 실질적 차이 없음)
