# cp·wrap 스킬 커밋 로직 스크립트화 개선

## 발단: 사용자 요청

cp 스킬 실행 시 레포 감지 오류(올바른 레포를 못 찾고 "커밋 없음" 종료) 발생. 자연어로 경로를 추론하다 생긴 문제. "모두 스크립트화할 수 있는 것들 아닌가?" 라는 지적에서 출발.

---

## 작업 상세내역

### 1. cp 레포 감지 오류 분석

**오류 원인**: cp 실행 시 변경 파일의 상위 경로 `D:/CloudSync`를 레포로 확인 → git 레포 없음 → 커밋 생략. 실제 레포는 `D:/CloudSync/05_AI/global-rule-improve`(하위 디렉토리).

cp 스킬 Step 1이 "변경 파일 dirname → git rev-parse" 로직을 자연어로 기술하여, Claude가 매번 경로를 추론해야 하는 구조였음.

### 2. cp Step 1 → find_save_target.js 스크립트화

**변경**: cp Step 1의 자연어 레포 감지 로직을 `find_save_target.js` 1회 호출로 대체.

```diff
- 세션에서 변경·생성한 파일들의 고유 디렉토리 목록 도출 → dirname → git rev-parse (자연어)
+ node ~/.claude/my-claude-plugins/shared/find_save_target.js <sessionId>
+ → git_repos[].root 를 커밋 대상으로 사용
```

### 3. find_save_target.js에 files[] 추가

**문제**: 스크립트가 레포별 `file_count`만 반환, 어느 파일이 어느 레포 소속인지 Claude가 추론.

**변경**: `git_repos[i]`에 `files[]` 추가. cp Step 2-2 staging도 `files[]` 직접 사용으로 변경.

```diff
- repoMap = Map<root, count>
+ repoMap = Map<root, files[]>
  → { root, files: [...], file_count }
```

### 4. find_save_target.js → shared/ 이동

**문제**: `~/.claude/skills/doc-save/scripts/`에 있었는데 cp도 쓰는 범용 유틸리티.

**변경**:
- 이동: `doc-save/scripts/find_save_target.js` → `my-claude-plugins/shared/find_save_target.js`
- 경로 업데이트: `doc-save/SKILL.md`, `cp/SKILL.md`
- 원본 trash 삭제

### 5. wrap Step 1 스코프 논의

**현황**: wrap Step 1이 `git diff --name-only HEAD` (CWD 기반 bash 루프)로 레포를 감지 → 다른 레포 변경 누락 가능성.

**결론**: wrap도 `find_save_target.js`로 교체해야 함 (다음 세션 작업).

### 6. wrap 커밋을 cp에 위임하는 방향 논의

**현황**: doc-save-cp = doc-save → cp 위임. 그런데 wrap은 독자적인 커밋 로직(Step 3)을 가짐 → cp와 불일치 발생.

**제안**: wrap Step 3도 cp에 위임(wrap = handoff → cp 위임)으로 단일 소스화.

**검토 결과**:
- wrap Step 3은 "관심사별 CHANGELOG 추가 → 1커밋씩 반복" 고유 로직 존재
- cp는 현재 레포당 1커밋 구조 → 멀티커밋 인터페이스 미지원
- 단순 위임 불가 → cp 인터페이스 확장 선행 필요

## 의사결정 기록

### cp 스코프 vs wrap 스코프

| | 레포 감지 | 커밋 범위 | 현재 상태 |
|---|---|---|---|
| doc-save | 세션 DB | 커밋 안 함 (저장만) | ✅ find_save_target.js 사용 |
| cp | 세션 DB | 레포 전체 미커밋 | ✅ find_save_target.js 사용 |
| wrap | git status (CWD) | 레포 전체 미커밋 | ⚠️ 수정 필요 |
| wrap (변경 후) | 세션 DB | 레포 전체 미커밋 | ⬜ 다음 세션 |

*cp는 세션 스코프로 레포를 찾되, 그 레포 내에서는 전체 미커밋 파일을 커밋 — 의도적 설계.*

### MSYS readFileSync 규칙 추가 검토 (이번 세션 앞부분)

- `readFileSync('/dev/stdin')` 패턴이 반복 발생 — MSYS가 경로 변환하여 ENOENT
- 규칙 추가 vs 미추가 검토 결과: **미추가 결정**
  - 빈발이지만 매 세션 규칙 로딩 비용이 시행착오(2~3회 retry) 비용보다 큼
  - 별도 문서화: `docs/20260327_01_MSYS-경로변환-readFileSync-규칙-검토.md`

## 논의해서 결정할 사안

### A. wrap 커밋 위임 시 cp 인터페이스 어떻게 확장할 것인가?

현재 cp는 "레포 내 전체 파일 → 1커밋". wrap은 "관심사별 그룹 → N커밋".

**옵션**:
1. cp에 `--split` 플래그 추가 — Claude가 파일을 관심사별로 그룹핑해서 여러 번 cp 호출
2. cp에 그룹핑 로직 내장 — diff 분석 후 자동 분류
3. wrap은 관심사 분류까지 담당, cp는 단건 커밋 전용으로 유지 → wrap이 cp를 N번 호출

### B. find_save_target.js 이름 변경 검토

현재 이름이 "저장 대상 찾기"라 doc-save 전용처럼 읽힘. `shared/`로 이동했으니 범용 이름 검토:
- `find_session_repos.js` 또는 `detect-session-repos.js`
