# 스킬 JSONL 직접 조회 → 세션DB 전환

## 작업 상세내역

### 배경

세션DB(`query-sessions.js`)는 속도·토큰 효율을 위해 도입됐으나, 세 스킬이 여전히 JSONL 파일을 직접 Read하고 있었다.

- `continue/SKILL.md` — DB hit 시 `get` + JSONL Read, DB miss 시 Claude 전용 Glob
- `history-insight/SKILL.md` — `find ~/.claude/projects/ -name "*.jsonl"` + JSONL 직접 파싱
- `session-analyzer/SKILL.md` — `find-session-files.mjs` + 하드코딩된 Claude 경로

### 변경 내역

| 스킬 | 변경 전 | 변경 후 |
|------|---------|---------|
| `continue` A-1 | `get` + JSONL Read (2단계) | `doc --no-sync` 1회 통합 |
| `continue` A-2 | Claude 전용 Glob | `codex:`/`gemini:`/UUID 멀티AI 분기 |
| `history-insight` Step 2 | `find` + `stat` (OS별 분기) | `by-project`/`recent` DB 쿼리 |
| `history-insight` Step 3 | JSONL Read / `extract-session.mjs` | `doc --no-sync` |
| `history-insight` Data Location | JSONL 경로 하드코딩 | 세션DB 안내로 교체 |
| `session-analyzer` Phase 1 | `find-session-files.mjs` | `query-sessions.js get` (`file_path` 획득) |

### 구체적 변경 포인트

**continue A-1**: `get` 호출 + `file_path`로 JSONL Read → `doc --no-sync` 단일 호출. `title`·`project`는 doc 헤더에 포함되므로 별도 get 불필요.

**continue A-2**: DB miss 폴백 경로에 멀티AI 분기 추가. session_id 접두어(`codex:`, `gemini:`)로 Glob 패턴을 분기하여 Claude 전용 구조 탈피.

**history-insight**: `find`+`stat` OS별 분기 로직 제거. `timestamp`/`last_timestamp` DB 필드로 날짜 필터 대체. Error Handling에서 `extract-session.mjs` 항목 제거, Related Resources도 `SESSION-DB.md` 단일 참조로 교체.

**session-analyzer**: `find-session-files.mjs` 및 Scripts Reference 항목 제거. Phase 1에서 `query-sessions.js get`으로 `file_path` 획득 후 debug log 분석(Phase 3~6)은 기존 유지.

## 의사결정 기록

- **결정**: `doc --no-sync`를 세션 컨텍스트 복원의 기본 인터페이스로 채택
- **근거**: `get`(메타만) + JSONL Read(원문)의 2-hop → `doc` 1-hop으로 단순화. `doc`의 마크다운 출력이 title·project·대화내용을 통합 제공하므로 중복 호출 불필요.
- **트레이드오프**: DB miss 케이스(A-2)는 여전히 Glob+Read 유지 — DB row가 없으면 `doc` 호출 자체가 불가하므로 폴백 필수.

- Codex/Gemini 세션의 A-2 Glob 패턴은 실제 파일 구조 검증 미완료 (경로 존재만 확인)

## 1. continue SKILL.md

### A-1 (DB hit) 변경

**Before:**
```
1. query-sessions.js get <sessionId>
2. file_path로 JSONL Read
3. 컨텍스트 요약: title, project, keywords, last_message, JSONL 마지막 assistant
```

**After:**
```
1. query-sessions.js doc <sessionId> --no-sync
   - 출력: 마크다운 (title, project, 대화 내용 포함)
   - get 별도 호출 불필요 (doc 헤더에 title·project 포함)
2. 컨텍스트 요약: doc 마크다운 출력 기준
```

- `get` 호출 제거, `doc --no-sync` 1회로 통합
- JSONL Read 관련 문구 전부 삭제
- `--limit` 미지정 (기본값 10 사용)

### A-2 (DB miss, Glob 폴백) 변경

**Before:** Claude 전용 Glob `~/.claude/projects/*/{sessionId}.jsonl`

**After:** session_id 접두어로 AI 타입 분기

```
- codex: 접두어 → ~/.codex/sessions/**/*-<UUID>.jsonl
- gemini: 접두어 → ~/.gemini/tmp/*/chats/session-*.json (UUID 매칭)
- 순수 UUID → ~/.claude/projects/*/{sessionId}.jsonl (기존)
```

파일 발견 후 Grep+Read는 유지 (DB에 row가 없으므로 doc 사용 불가).
