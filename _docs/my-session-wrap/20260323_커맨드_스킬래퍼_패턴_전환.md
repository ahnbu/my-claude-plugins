# my-session-wrap 커맨드 스킬 래퍼 패턴 전환

## 발단: 사용자 요청

`history-insight` 스킬이 Claude Code에서 인식되지 않는 문제 진단 요청. 원인 분석 과정에서 플러그인 전체의 커맨드-스킬 구조 비일관성과 런타임 비동기 문제를 발견하여 체계적 정리로 확대.

## 작업 상세내역

### 1단계: history-insight 미인식 원인 진단

- SKILL.md 파일은 소스 레포에 존재 확인
- 현재 "Plugin 미설치 운용" 모드에서 스킬 디스커버리는 `~/.claude/skills/`에서만 수행
- `history-insight`가 `~/.claude/skills/`에 미배치 → 인식 불가 확인

### 2단계: 커맨드-스킬 구조 전수 점검

기존 인식 상태 전수 조사:

| 항목 | 인식 | 경유 |
|------|:----:|------|
| `wrap` | ✅ | `~/.claude/commands/wrap.md` (복사본, 비동기) |
| `continue` | ✅ | `~/.claude/commands/continue.md` (복사본, `plugin:` 마커 누락) |
| `save` | ✅ | `~/.claude/commands/save.md` (복사본, `plugin:` 마커 누락) |
| `cp` | ✅ | `~/.claude/commands/cp.md` (복사본, 내용 비동기 — 멀티레포 로직 구버전) |
| `session-find` | ✅ | `~/.claude/skills/session-find/` (복사본) |

추가 발견:
- `ensure-commands.js` (SessionStart 자동 동기화)가 플러그인 미활성으로 **실행되지 않고 있음**
- `enabledPlugins`에 `my-session-wrap` 미등록
- Codex/Gemini는 Junction으로 소스 SKILL.md 직접 참조 → 동기화 문제 없음

### 3단계: 스킬 래퍼 패턴 설계 및 적용

**패턴**: 커맨드 `.md`를 얇은 래퍼로 만들고, 소스 레포의 SKILL.md 절대경로를 Read하도록 지시.

```
commands/wrap.md (래퍼, 5줄)
  → "Read and follow: .../skills/my-session-wrap/SKILL.md"
     → SKILL.md (정본, 310줄)
```

장점:
- 래퍼 자체는 거의 변경 불필요 → 런타임 복사본 비동기 문제 구조적 해소
- SKILL.md 수정 즉시 반영 (Read 시점에 최신 파일 로드)
- Codex/Gemini는 기존 Junction 유지 (변경 불필요)

현재 세션(d38c0d1c)을 대상으로 두 스킬을 실행하여 정상 동작 확인:

두 스킬 모두 Node.js CLI 기반으로 정상 동작. 커맨드 등록 없이 스크립트 직접 호출로 사용 가능.

### 5단계: 스킬 체이닝 커맨드 (`/doc-save-cp`)

`/doc-save` 후 `/cp`를 연속 실행하는 패턴이 반복되어, 복합 래퍼 커맨드를 생성.

- **배치**: `~/.claude/commands/doc-save-cp.md` (글로벌 커맨드로만 관리)
- **소스 레포에는 미포함**: `doc-save`와 `cp`는 별개 스킬이므로 `my-session-wrap` 플러그인의 고유 기능이 아님
- Claude Code에 공식 스킬 체이닝 문법은 없으며, `&&`이나 `->` 같은 표기는 AI가 의도를 해석하는 것에 의존

## 의사결정 기록

### 커맨드 래퍼 vs 스킬 복사 vs Junction/Hardlink

| 방식 | 동기화 | 유지비용 | 제약 |
|------|:------:|:--------:|------|
| **스킬 래퍼 커맨드** ✅ | 즉시 반영 | 낮음 | 래퍼→런타임 1회 복사 필요 |
| 스킬 폴더 복사 | 수동 동기화 | 높음 | 수정마다 복사 필요 |
| Junction | 즉시 반영 | 낮음 | 디렉토리 전용, 개별 파일 불가 |
| Hardlink | 즉시 반영 | 낮음 | git 파일 교체 시 링크 끊김 |

- **결정**: 스킬 래퍼 커맨드 패턴 채택
- **근거**: 래퍼는 거의 바뀌지 않아 1회 복사로 충분. SKILL.md 수정이 즉시 반영되어 이중 관리 해소
- **트레이드오프**: 래퍼가 바뀌면 수동 복사 필요하지만, 래퍼 변경 빈도가 극히 낮아 실용적

### `allowed-tools` 제거

- **결정**: 커맨드 frontmatter에서 `allowed-tools` 제거
- **근거**: 전역 설정에서 도구 권한을 이미 관리. 커맨드별 별도 관리는 유지포인트만 늘리고, Agent 도구 누락 등 오히려 기능 제약 유발

### `save` 커맨드 폐기

- **결정**: `save` 커맨드/스킬을 `temp/`로 이동 (미사용)
- **근거**: 사용자가 사용하지 않는다고 명시
