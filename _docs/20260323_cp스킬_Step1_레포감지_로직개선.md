# (Plan) /cp 스킬 Step 1 레포 감지 로직 개선

## Context

`/cp` 실행 시 CWD만 스캔하여, CWD가 git 레포가 아니거나 변경 파일이 다른 디렉토리에 있으면 감지 실패. 예: `D:\CloudSync` 하위 git 레포의 변경이 누락됨. 추가로 cp 관련 파일이 유령 복사본과 잘못된 junction으로 분산되어 있음.

## 현재 파일 구조 및 문제

```
[플러그인 원본] ← 수정 대상
my-claude-plugins/my-session-wrap/skills/cp/SKILL.md
my-claude-plugins/my-session-wrap/commands/cp.md  (→ 스킬 리다이렉트)

[Claude 런타임]
~/.claude/commands/cp.md          ← ensure-commands 복사 (OK)
~/.claude/skills/cp/SKILL.md      ← ⚠️ 유령 파일 (삭제 대상)

[Junction - 잘못된 참조]
~/.gemini/skills/cp/ → ~/.claude/skills/cp/   ← 유령 파일 참조 (재설정 대상)
~/.codex/skills/cp/  → ~/.claude/skills/cp/   ← 유령 파일 참조 (재설정 대상)
```

## 작업 목록

### 1. 유령 파일 삭제 및 junction 재설정

1. `~/.claude/skills/cp/` 삭제 (유령 파일)
2. Gemini junction 재설정: `~/.gemini/skills/cp/` → `my-claude-plugins/my-session-wrap/skills/cp/`
3. Codex junction 재설정: `~/.codex/skills/cp/` → `my-claude-plugins/my-session-wrap/skills/cp/`

### 2. Step 1 로직 개선

**수정 파일**: `my-claude-plugins/my-session-wrap/skills/cp/SKILL.md` — Step 1만 교체

**현재 문제**: CWD에서 `git diff`/`git ls-files` 실행 → CWD가 git 레포가 아니면 파일 수집 실패

**개선 방향**: 변경 파일 기준으로 레포를 찾는다 (디렉토리 전수 스캔 X)

1. Claude는 세션에서 변경/생성한 파일 경로를 이미 알고 있음
2. 각 파일의 `dirname`에서 `git rev-parse --show-toplevel` → 소속 레포 루트 감지
3. git 레포에 속하지 않는 파일은 안내 후 스킵
4. 감지된 레포별로 `git status --porcelain`으로 전체 변경 파일 확인 후 그룹핑

**Step 1 교체 내용** (기존 18~46행):

```markdown
### Step 1: 레포 감지 및 그룹핑

세션에서 변경/생성한 파일 경로를 기준으로 소속 git 레포를 감지하고 그룹핑한다.

#### 1-1. 변경 파일 기준 레포 감지

세션에서 변경·생성한 파일들의 **고유 디렉토리 목록**을 먼저 도출한 뒤, 각 디렉토리에서 레포를 찾는다:

1. 변경 파일 경로 → `dirname` → 중복 제거 → 고유 디렉토리 set
2. 각 디렉토리에서 `git rev-parse --show-toplevel` → 레포 루트 감지
3. 레포 루트 중복 제거

\```bash
# 고유 디렉토리별 1회만 실행
git -C "<디렉토리>" rev-parse --show-toplevel 2>/dev/null
\```

- git 레포에 속하지 않는 디렉토리 → "git 레포 아님 — 스킵" 안내

#### 1-2. 레포별 변경 확인

각 레포에서 전체 변경 파일을 확인한다:

\```bash
git -C "<REPO>" status --porcelain
\```

그룹핑 결과를 사용자에게 표시:
\```
📂 감지된 레포 (N개):
- <레포루트A> (M files)
- <레포루트B> (K files)
\```

변경사항이 전혀 없으면 "커밋할 내용이 없습니다." 출력 후 종료.
```
