# Plan: copyDoc 2모드 분리 (기본 / 상세)

## Context

`/ss` 대시보드의 "문서 전체 복사" 버튼이 tool 호출 정보를 완전히 제외하고 복사한다. 사용자는 화면에 보이는 그대로 복사되길 원하되, 상황에 따라 tool 상세 정보 포함 여부를 선택하고 싶어함.

## 현재 동작 (문제)

`copyDoc` (index.html:1561-1568):
- `m.text`만 포함, `m.tools` 무시
- text 없이 tool만 있는 메시지 완전 누락
- tool_result 메시지도 "User"로 표시 (구분 없음)

## 변경 계획

**파일:** `my-session-dashboard/index.html`

### 1. 버튼 UI 변경 — 2개 버튼으로 분리

기존 "문서 전체 복사" 버튼 1개 → 2개로:

| 버튼 | 라벨 | 동작 |
|------|------|------|
| 기본 복사 | `문서 복사` | text + 역할 라벨 분화. tool은 한 줄 요약만 |
| 상세 복사 | `문서 복사 (상세)` | 기본 + tool input 전체 (접힘 블록 내용까지 포함) |

### 2. 공통 변경 — 역할 라벨 분화

- `user` + `subtype === "meta"` → `스킬`
- `user` + `subtype === "tool_result"` → `도구 응답`
- `user` (기타) → `User`
- assistant → `Assistant`

### 3. 기본 모드 (tool 요약)

- text가 있으면 text 출력
- tools가 있으면 `> 🔧 **{name}** {toolSummary(tool)}` 한 줄씩 추가
- text도 tools도 없으면 제외

### 4. 상세 모드 (tool input 포함)

- 기본 모드와 동일 + 각 tool 아래에 JSON input 추가

### 5. 구현 — copyDoc 함수 리팩터

`copyDoc(btn, sessionId)` → `copyDoc(btn, sessionId, detailed=false)` 파라미터 추가.
`messagesToMarkdown(messages, detailed)` 헬퍼 함수 추출하여 두 모드 공유.

### 재사용 함수

- `toolSummary(tool)` (1462행) — 그대로 사용
- `formatToolInput(tool)` (1453행) — 상세 모드에서 참고

## 검증 방법

1. 정적 HTML이므로 빌드 불필요
2. `/ss` 대시보드에서 tool 호출 포함 세션 선택
3. "문서 복사" → tool 한 줄 요약만 포함 확인
4. "문서 복사 (상세)" → tool input JSON까지 포함 확인
5. tool만 있고 text 없는 메시지도 양쪽 모두 포함되는지 확인
6. tool_result 메시지가 "도구 응답"으로 표시되는지 확인
