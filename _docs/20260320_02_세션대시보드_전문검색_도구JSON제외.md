# 세션 대시보드 전문 검색 — 도구 JSON 제외, 대화 텍스트 전체 포함

## 발단: 사용자 요청

1차 성능 최적화에서 검색 인덱스에 대화 텍스트 첫 2,000자만 포함 → 대화 후반부 검색 불가.
사용자 피드백: "전문 검색은 중요한 기능이다. 2,000자는 일부만 가능하다."

## 작업 상세내역

### 97MB 데이터 구성 분석

| 구성 요소 | 크기 | 비율 | 검색 가치 |
|-----------|------|------|----------|
| 도구 JSON (input/output) | 46MB | 69% | ❌ 낮음 |
| AI 응답 텍스트 | 12MB | 19% | ✅ 높음 |
| 사용자 프롬프트 | 5MB | 7% | ✅ 높음 |
| 스킬/메타 텍스트 | 3MB | 5% | 보통 |
| 도구 응답 텍스트 | ~0MB | 0% | 보통 |

### 변경 내용

`build.js` 검색 인덱스 생성 로직 수정:
- `substring(0, 2000)` 제한 제거
- `m.tools` (도구 JSON) 제외, `m.text`만 포함
- plan 콘텐츠는 전체 포함 (기존 동일)

### 예상 효과

| 지표 | 현재 (2,000자 제한) | 변경 후 (전체 텍스트) |
|------|-------------------|---------------------|
| 검색 인덱스 크기 | ~6MB | ~20MB |
| HTML 크기 | 8.3MB | ~22MB |
| 전문 검색 | ❌ 후반부 불가 | ✅ 도구 JSON 제외 전체 |
| 초기 로드 | <1초 | ~2초 |

## 의사결정 기록

- 결정: 도구 JSON 제외, 대화 텍스트 전체 인덱싱
- 근거: 도구 JSON이 69%(46MB)로 검색 가치 낮음. 제외 시 ~20MB로 99MB 대비 78% 감소. 검색 속도 <200ms 유지.
- 트레이드오프: HTML 8.3MB→~22MB, 초기 로드 <1초→~2초. 99MB/10초 대비 크게 개선된 수준.
