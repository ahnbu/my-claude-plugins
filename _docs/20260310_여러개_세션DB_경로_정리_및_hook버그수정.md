## 의사결정 기록

### 핵심 결정: marketplace DB를 primary로

| 옵션 | 장점 | 단점 |
|------|------|------|
| ✅ marketplace primary | hooks·build.js 모두 marketplace 기록, 일관성 확보 | 소스 레포 DB가 stale 됨 |
| 소스 레포 primary | hook 하드코딩 수정 불필요 | build.js와 hooks가 다른 DB 기록, 분기 지속 |

- **결정**: marketplace primary
- **근거**: hooks의 `__dirname` 수정 후 모든 컴포넌트가 marketplace DB에 기록 → single source of truth 확립
- **트레이드오프**: 소스 레포 DB(57MB)의 기존 데이터는 marketplace DB(75MB)에 포함되지 않으나, 향후 신규 데이터는 marketplace에만 쌓임

### 코드 분석 결과

- `session-loader.js`: `os.homedir()` 기반 → 소스 레포 DB에서 읽기
