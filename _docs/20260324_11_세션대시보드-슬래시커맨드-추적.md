# (Plan) 세션 대시보드: 슬래시 커맨드(스킬) 사용 이력 추적

## Context

사용자가 `/wrap`, `/doc-act`, `/continue` 등 슬래시 커맨드(스킬)를 호출하면, JSONL에 `<command-name>/wrap</command-name>` 태그로 기록된다. 그러나 `stripSystemTags()`가 이 태그를 노이즈로 제거하고, 별도 추출/집계 로직이 없어 **세션 대시보드에서 스킬 사용 이력이 완전히 유실**된다.

빌트인 커맨드(`/init` 등)와 스킬(`/wrap` 등) 구분 없이 통합 집계한다.

## 수정 파일 및 변경 내용

### 1. `shared/text-utils.js` — 커맨드명 추출 함수 추가

- 새 함수 `extractSlashCommands(text)` 추가
- `<command-name>` 태그에서 커맨드명 배열 반환
- `stripSystemTags()` 호출 **전에** 실행해야 함

```js
function extractSlashCommands(text) {
  if (!text) return [];
  const re = /<command-name>([^<]+)<\/command-name>/gi;
  const cmds = [];
  let m;
  while ((m = re.exec(text)) !== null) {
    cmds.push(m[1].trim());
  }
  return cmds;
}
```

### 2. `shared/session-parser.js` — processSession() 내 집계 추가

- 변수 선언부(~L241): `const slashCommands = [];` 추가
- user entry 처리부(~L266): `stripSystemTags()` 호출 전에 `extractSlashCommands(rawText)` 호출하여 `slashCommands`에 push
- metadata 객체(~L354): `slashCommands` 필드 추가

```js
// ~L241
const slashCommands = [];

// ~L266, stripSystemTags 호출 전
const cmds = extractSlashCommands(rawText);
slashCommands.push(...cmds);

// ~L370 metadata 객체
slashCommands,
```

### 3. `shared/session-db.js` — DB 스키마 및 upsert 수정

- sessions 테이블에 `slash_commands TEXT` 컬럼 추가 (~L100, `originator` 뒤)
- `_upsertSession()`의 INSERT 컬럼 목록과 VALUES에 `slash_commands` 추가
- 값: `JSON.stringify(metadata.slashCommands || [])`
