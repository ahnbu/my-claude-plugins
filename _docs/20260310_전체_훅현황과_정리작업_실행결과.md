## 작업 상세내역

### 작업 1: check-handoff.js 비활성화

**before** (`settings.json` SessionStart 섹션):

→ 섹션 전체 삭제

### 작업 2: pre-commit 버전 동기화 훅 비활성화

**목적**: `marketplace.json` ↔ `plugin.json` 버전 불일치 시 커밋 차단. 미설치 모드에서 배포를 하지 않으므로 불필요.

**변경 내용**:

**목적**: 3개 훅 이벤트(SessionStart/UserPromptSubmit/Stop)에서 세션 ID를 파일로 저장해 이벤트별 비교하는 테스트용 플러그인. `settings.json`에 미등록 상태였고, `my-session-wrap`의 `capture-session-id.js`(stdout 주입 방식)로 완전 대체됨.

**변경 내용**:

## 리스크 및 미해결 이슈

1. **hooks.json 깨진 참조 2건** — `my-session-wrap/hooks.json`이 `check-handoff.js`, `inject-plugin-guidelines.js`를 참조하나 실제 파일은 `.bak`. 설치 모드 전환 시 훅 미실행 상태로 전환됨. 수정 필요.
2. **`.git/hooks/post-commit` 잔존** — `core.hooksPath`로 무시되고 참조 파일도 없어 무해하나, 혼동 소지 있음. 삭제 검토 가능.
3. **`check-version-sync.js.bak` 이중 상태** — 파일명은 `.bak`(비활성 의도)이지만 `git-hooks/pre-commit`이 `node "$(dirname "$0")/check-version-sync.js.bak"`으로 명시 호출하여 사실상 활성. 파일명을 원래대로 복원하거나 pre-commit 호출 경로를 수정해야 일관성 확보.
4. **CLAUDE.md "pre-commit hook이 검증" 문구** — 실제 담당 훅이 `git-hooks/pre-commit`(이 레포 전용)과 글로벌 pre-commit(모든 레포)으로 분리되어 있음. CLAUDE.md의 설명이 단순화되어 있어 오해 가능성 있음. 향후 문서 정밀화 검토.
