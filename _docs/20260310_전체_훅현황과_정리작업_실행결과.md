## 발단: 사용자 요청

Option C(Plugin 미설치 운용) 전환 완료 후, 미설치 모드에서 불필요하거나 사용 빈도가 낮은 훅 3종을 정리하는 계획을 실행하라.

- **작업 1**: `check-handoff.js` 비활성화 (사용 빈도 낮음)
- **작업 2**: `pre-commit` 버전 동기화 훅 비활성화 (미설치 모드에서 배포 불필요)
- **작업 3**: `my-session-id` 플러그인 삭제 (초기 테스트용, 현재 미사용)

## 작업 상세내역

### 작업 1: check-handoff.js 비활성화

**목적**: 24시간 내 handoff 감지 시 `/continue` 안내를 출력하는 SessionStart 훅. 사용 빈도 낮아 비활성화.

**변경 내용**:
- `~/.claude/settings.json` — `"SessionStart"` 블록 전체 제거 (유일한 항목이었으므로 key 포함 제거)
- `my-session-wrap/hooks/check-handoff.js` → `check-handoff.js.bak` (배포 복원용 보존)

**before** (`settings.json` SessionStart 섹션):
```json
"SessionStart": [
  {
    "hooks": [
      {
        "type": "command",
        "command": "node C:/Users/ahnbu/.claude/my-claude-plugins/my-session-wrap/hooks/check-handoff.js",
        "timeout": 3000
      }
    ]
  }
],
```
→ 섹션 전체 삭제

### 작업 2: pre-commit 버전 동기화 훅 비활성화

**목적**: `marketplace.json` ↔ `plugin.json` 버전 불일치 시 커밋 차단. 미설치 모드에서 배포를 하지 않으므로 불필요.

**변경 내용**:
- `git-hooks/pre-commit` → `pre-commit.bak`
- `git-hooks/check-version-sync.js` → `check-version-sync.js.bak`
- `.git/hooks/pre-commit` 삭제 (git이 참조하던 실행 파일)

### 작업 3: my-session-id 플러그인 삭제

**목적**: 3개 훅 이벤트(SessionStart/UserPromptSubmit/Stop)에서 세션 ID를 파일로 저장해 이벤트별 비교하는 테스트용 플러그인. `settings.json`에 미등록 상태였고, `my-session-wrap`의 `capture-session-id.js`(stdout 주입 방식)로 완전 대체됨.

**변경 내용**:
- `my-session-id/` 폴더 전체 삭제
- `CLAUDE.md` — 현재 플러그인 테이블에서 `my-session-id` 행 제거
- `.claude-plugin/marketplace.json` — `my-session-id` 항목 제거
- `README.md` — 디렉토리 트리, 설치 명령어, 플러그인 설명 섹션, 설치범위 표, 업데이트 구조에서 `my-session-id` 참조 5곳 제거
- `CHANGELOG.md` — 이번 작업 3건 기록 추가 (최상단)

## 의사결정 기록

| 항목 | 채택안 | 근거 |
|------|--------|------|
| check-handoff.js | `.bak` 보존 | 배포 시 재활성화 가능성 남김 |
| pre-commit | `.bak` 보존 | 배포 시 버전 검증 로직 재사용 예정 |
| my-session-id | 완전 삭제 | settings.json 미등록, my-session-wrap이 완전 대체, 복원 필요성 없음 |
| hooks.json 파일들 | 유지 | Plugin 설치 모드 전환 시 그대로 동작. 현재 무해하므로 삭제 불필요 |
| CHANGELOG.md 과거 기록 | 유지 | 이력 보존 원칙 |

- **결정**: 불필요 훅은 `.bak` 리네임(복원 가능), 불필요 플러그인은 완전 삭제
- **트레이드오프**: 배포 전환 시 `.bak` 복원 1단계 추가되나, 현재 세션 오버헤드 제거 우선
