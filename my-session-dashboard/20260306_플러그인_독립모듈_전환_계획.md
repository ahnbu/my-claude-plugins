### 단계별 작업

#### 1. SKILL.md 이동
- `my-session-dashboard/skills/session-dashboard/SKILL.md` → `~/.claude/skills/session-dashboard/SKILL.md`
- 경로 참조를 `my-session-dashboard/` 절대 경로로 업데이트

#### 2. build.js DIST_DIR 수정 (L12)
```js
// 변경 전
const DIST_DIR = path.join(__dirname, "..", "output", "session-dashboard");
// 변경 후
const DIST_DIR = path.join(__dirname, "output");
```

#### 3. serve.js OUTPUT_HTML 수정 (L15~17)
```js
// 변경 전
const OUTPUT_HTML = path.join(PLUGIN_DIR, '..', 'output', 'session-dashboard', 'index.html');
// 변경 후
const OUTPUT_HTML = path.join(PLUGIN_DIR, 'output', 'index.html');
```

#### 4. 플러그인 보일러플레이트 삭제
- `.claude-plugin/` 폴더 전체
- `hooks/` 폴더 전체
- `commands/` 폴더 전체
- `skills/` 폴더 전체 (이동 완료 후)

#### 5. `~/.claude/commands/ss.md` 삭제
- hook이 복사한 파일. 스킬 트리거가 대체.

#### 6. marketplace 복사본 정리
- `~/.claude/plugins/marketplaces/my-claude-plugins/my-session-dashboard/` 내부에서 session-dashboard 관련 파일 삭제

#### 7. 기존 외부 output 삭제
- `~/.claude/my-claude-plugins/output/session-dashboard/` 삭제

## 보충: 대화 중 검토 내용

### Plan 변천 과정

#### v1 — "output 경로 통일 + serve.js SSE 버그 수정" (기술 버그 수정 관점)

버그 증상:
- 기존 HTML 즉시 로딩 안 됨 (로딩 페이지 표시)
- `ERR_CONNECTION_REFUSED` SSE 연결 실패

기술적 원인:
- `build.js`: `path.join(__dirname, "..", "output", "session-dashboard")` → 플러그인 외부 출력
- `serve.js`: `path.join(PLUGIN_DIR, 'output', 'index.html')` → 플러그인 내부 탐색
- 두 경로 불일치

v1 plan: output을 플러그인 내부로 통일하면 해결 → 단순 경로 수정 2줄

#### v2 (현재) — "플러그인 구조 자체 재검토" (아키텍처 관점)

사용자 근본 질문:
> "굳이 플러그인으로 해야 하는 이유가 있는가? 여러모로 불편한 것 같다. 결국 내가 쓰자고 하는 건데. 배포하는 것도 아니고. 자동 업데이트가 된다는 건데, 사실 내가 만드는 것이니까 이것은 장점이 아니다."

플러그인이 실제로 제공하는 것 분석 결과:
- 유일한 기능: `commands/ss.md`를 `~/.claude/commands/`에 자동 복사
- 하지만 `SKILL.md`에 `Triggers: '/ss'` 이미 있어서 스킬 트리거가 동일하게 동작 → **hook 불필요**

v1 수정 방향을 버리고 플러그인 구조 제거 + 독립 프로젝트 전환으로 확대

### 플러그인 제공 가치 vs 비용 분석

#### session-dashboard 플러그인이 제공하는 것

| 기능 | 내용 | 실제 가치 |
|---|---|---|
| SessionStart hook | `commands/ss.md`를 `~/.claude/commands/`에 자동 복사 | ❌ 불필요 — `SKILL.md` Triggers로 대체 가능 |
| plugin.json 메타데이터 | name, version, description | ❌ 개인 도구에 불필요 |
| marketplace 배포 | 자동 업데이트 | ❌ 본인 개발이므로 불필요 |
| 커맨드 충돌 감지 | `plugin:` 마커 소유권 판별 | ❌ 본인만 사용 |

#### 플러그인의 비용

| 비용 | 내용 |
|---|---|
| 이중 관리 | 소스(`my-claude-plugins/`) + marketplace(`plugins/marketplaces/`) 매번 동기화 |
| 경로 혼동 | `../output/`, `$CLAUDE_PLUGIN_ROOT` 등 간접 경로 → 방금 겪은 버그의 원인 |
| 보일러플레이트 | `plugin.json`, `hooks.json`, `ensure-commands.js` (실 기능 없음) |

### 전체 플러그인 Hook 상세 분석

#### my-session-wrap (플러그인 유지 필요)

| Hook | 이벤트 | 기능 | 스킬 대체 가능? |
|---|---|---|---|
| `capture-session-id.js` | UserPromptSubmit | 매 프롬프트마다 세션ID를 AI 컨텍스트에 자동 주입 | ❌ 스킬은 사용자 호출 필요 |
| `check-handoff.js` | SessionStart | 24시간 이내 handoff 파일 감지 → 자동 안내 | ❌ 사용자 호출 없이 실행 |
| `inject-plugin-guidelines.js` | SessionStart | 특정 디렉토리 감지 후 경고 자동 주입 | ❌ 동일 이유 |
| `ensure-commands.js` | SessionStart | 커맨드 자동 배포 | ⚠️ 수동 복사로 대체 가능 |

#### my-session-id (플러그인 유지 필요)

| Hook | 이벤트 | 기능 |
|---|---|---|
| `capture-session-id.js` | SessionStart / UserPromptSubmit / Stop | 세션 시작·프롬프트·종료 시 세션ID 파일 기록 |

hook 3개가 핵심 기능 자체 → 플러그인 없으면 동작 불가

#### my-session-dashboard (독립 전환 가능)

| Hook | 이벤트 | 기능 | 대체 방법 |
|---|---|---|---|
| `ensure-commands.js` | SessionStart | `ss.md` 자동 복사 | SKILL.md `Triggers: '/ss'`로 대체 |

hook이 편의 기능일 뿐, 핵심 기능(build.js, serve.js)은 Node.js 파일 → 어디 있든 무관
