## 변경 범위

### 변경 1: Step 1 확장 — 멀티 레포 감지 추가

기존 Step 1 (`git status --short`)에 멀티 레포 감지 로직을 추가한다:

```
Step 1. Git 감지 + 멀티 레포 그룹핑
```

1. 기존: `git status --short` 실행
2. **추가**: 세션에서 변경된 파일 목록을 수집하고, 각 파일에 대해 `git rev-parse --show-toplevel`로 레포 루트 감지
3. 레포 루트별로 파일 그룹핑
4. 그룹핑 결과 표시

### 변경 2: Step 3-1 수정 — CHANGELOG.md 위치 확인을 레포별로

기존: handoff 경로에서 ProjectRoot 역산 → 1개 CHANGELOG만 확인
변경: 각 레포 루트에서 CHANGELOG.md 존재 여부를 개별 확인

### 변경 3: Step 3-2 수정 — 관심사별 커밋을 레포별로 순회

기존:
```
관심사 A → CHANGELOG 1줄 + git add + commit
관심사 B → CHANGELOG 1줄 + git add + commit
```

변경:
```
📂 레포 A:
  관심사 A-1 → CHANGELOG 1줄 + git -C <레포A> add + commit
  관심사 A-2 → CHANGELOG 1줄 + git -C <레포A> add + commit

📂 레포 B:
  관심사 B-1 → CHANGELOG 1줄 + git -C <레포B> add + commit
```

- 관심사 분류 시 레포별로 그룹핑한 뒤, 레포 내에서 관심사별 분리 커밋 수행
- `git -C <레포루트>` 형식으로 레포를 명시

## 보충: 대화 중 검토 내용

### 의사결정 근거

**후속 B (cp 위임) 보류 이유:**
- 스킬 간 직접 호출 불가 (Claude Code 구조적 제약)
- 커밋 철학 차이: cp는 레포별 1커밋, wrap은 관심사별 분리 커밋 — 동일 로직으로 통합 불가

**멀티 레포 감지에 `git rev-parse --show-toplevel` 사용 이유:**
- 각 파일 경로에서 해당 파일이 속한 git 레포 루트를 정확히 반환
- cross-drive 환경(D:/, C:/)에서도 정상 동작
- `git -C <경로>` 옵션으로 특정 레포에 명시적으로 작업 가능

**handoff 파일 1개 유지 이유:**
- handoff는 세션 단위 컨텍스트 문서 — 레포별 분리 불필요
- 여러 레포에 걸친 작업도 하나의 세션으로 관리
