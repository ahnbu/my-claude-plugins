> 저장일: 2026-03-07 | 원본: C:/Users/ahnbu/.claude/plans/cozy-popping-honey.md

## 개요

현재 `/cp` 스킬은 CWD 기준 단일 git 레포에서만 동작한다. Claude Code 세션에서 `additionalDirectories`에 등록된 여러 프로젝트 파일을 수정한 뒤 `/cp`를 실행하면, CWD가 아닌 레포의 변경사항은 누락된다.

**목표**: 변경된 파일들을 각 파일이 속한 git 레포 루트별로 자동 그룹핑하여, 레포마다 개별 commit + CHANGELOG 업데이트를 수행하도록 개선한다.

## 구현 계획

### Step 1: 상태 확인 (기존 Step 1 대체)

기존 CWD 기준 `git status` 대신:

1. 세션에서 변경된 파일 목록 수집 (현재 대화 컨텍스트 기반)
2. 각 파일에 대해 `git -C <파일경로> rev-parse --show-toplevel`로 레포 루트 감지
3. 레포 루트별로 파일 그룹핑
4. git 레포에 속하지 않는 파일은 별도 안내 후 스킵
5. 그룹핑 결과를 사용자에게 표시:
   ```
   📂 감지된 레포 (2개):
   - C:/Users/ahnbu/.claude/my-claude-plugins (3 files)
   - C:/Users/ahnbu/project-x (1 file)
   ```

### Step 2: 레포별 순회 처리

각 레포에 대해 기존 Step 2~5를 순차 실행:

1. **Staging**: 해당 레포의 변경 파일만 `git -C <레포루트> add <파일명...>`
2. **커밋 메시지 생성**:
   - `git -C <레포루트> diff --cached` 분석
   - **작업 맥락 반영 가이드**: 여러 레포에 걸친 작업인 경우, 커밋 메시지에 전체 작업 맥락을 포함 (예: "플러그인 A 변경에 따른 설정 동기화")
3. **CHANGELOG 업데이트**: 해당 레포 루트의 `CHANGELOG.md` 존재 시 업데이트
4. **Commit**: `git -C <레포루트> commit ...`
5. **Push**: 해당 레포에 remote가 있으면 push

### Step 3: 에러 처리

- 특정 레포에서 커밋 실패 시: 에러 메시지 출력 후 다음 레포로 계속 진행
- 최종 결과에서 성공/실패 레포를 구분하여 표시

```
멀티 레포 커밋 메시지 작성 시:

- 다른 레포 변경과 연관된 경우, 커밋 메시지 본문에 관련 맥락 기술

```

## 후속 작업 (본 작업 완료 후)

### 후속 A: /wrap 스킬 멀티 레포 개선

`/wrap` Step 3 (git commit)도 현재 단일 레포 기준이다. `/cp`와 동일한 멀티 레포 감지·분리 패턴을 `/wrap`에도 적용.

**현재 /wrap Step 3 동작**:
- `next-handoff.sh`가 반환한 handoff 경로에서 ProjectRoot 역산
- 해당 ProjectRoot 기준으로만 CHANGELOG + 관심사별 커밋 수행
- 다른 레포의 변경사항은 처리하지 않음

**개선 방향**:
- Step 1에서 모든 변경 파일의 레포 루트를 감지·그룹핑
- Step 3에서 레포별로 CHANGELOG + 관심사별 커밋 순회
- handoff 파일은 기존대로 하나만 생성 (세션 단위 문서)

### 후속 B: /wrap의 커밋·CHANGELOG를 /cp에 위임

/wrap Step 3의 커밋 로직을 `/cp` 호출로 대체하여 중복 제거.

| 항목 | 현행 (/wrap 자체 커밋) | 위임안 (/cp 호출) |
|------|----------------------|-------------------|
| 코드 중복 | ❌ /cp와 커밋 로직 이중 관리 | ✅ /cp 한 곳에서 관리 |
| 유지보수 | ⚠️ 양쪽 동기화 필요 | ✅ /cp만 수정하면 /wrap에 자동 반영 |
| 관심사 분리 | ████░░ /wrap이 모든 것 담당 | ✅ /wrap=handoff, /cp=커밋 |
| 관심사별 분리 커밋 | ✅ /wrap이 직접 제어 | ⚠️ /cp가 이 기능을 지원해야 함 |
| 실행 흐름 제어 | ✅ 직접 제어 | ⚠️ /cp 스킬 호출 방식 확인 필요 |

**핵심 고려사항**: 현재 `/wrap`은 관심사별 분리 커밋(1 CHANGELOG 행 + 관련 파일 → 1 commit)을 수행한다. `/cp`에 위임하려면 `/cp`도 이 관심사 분리 로직을 지원해야 한다. 또는 `/wrap`이 관심사 분류까지만 하고, 레포별 커밋 실행만 `/cp`에 위임하는 하이브리드 방식도 가능.
